# Gemini 阶段开发指挥文档

本文档用于直接复制给 Gemini。每次只复制当前阶段，不要一次复制全部阶段让 Gemini 同时开发。

## 通用开场指令

```text
你现在是 Android 原生开发助手，需要基于“口袋星球 / Planet Life”项目进行开发。

统一要求：
1. 使用 Kotlin。
2. 使用 Jetpack Compose。
3. 使用 MVVM。
4. 使用 Navigation Compose 做页面跳转。
5. 使用 Room 做核心数据持久化。
6. 使用 DataStore 或 SharedPreferences 保存简单设置。
7. MVP 不做服务器、登录、云同步、复杂 3D、内购、AI 日报。
8. 不要把业务逻辑写在 Composable 里。
9. 不要让 UI 直接操作数据库。
10. 每次开发完成后列出新增/修改文件、实现内容、验证命令、已知问题。
11. 必须保证项目可以在 Android Studio 中打开并运行。
12. UI 必须参考 docs/assets/ui-reference 目录中的图片：浅蓝天空背景、像素风、奶油色面板、五栏底部导航。
13. 底部导航固定为：星球、日志、专注、任务、我的。图鉴从“我的”进入，不放入 MVP 底部导航。
```

## 阶段 0 指令：项目初始化

```text
阶段目标：
创建一个可以运行的 Android 原生项目骨架。

请完成：
1. 创建 Android Kotlin + Jetpack Compose 项目，应用名为“口袋星球”。
2. 包名建议使用 com.example.planetlife。
3. 配置 Navigation Compose。
4. 配置 Room。
5. 配置 DataStore，如果依赖处理麻烦，可先使用 SharedPreferences，但要保留后续替换空间。
6. 建立以下目录结构：
   - navigation
   - data/local/entity
   - data/local/dao
   - data/repository
   - data/settings
   - domain/rules
   - ui/home
   - ui/focus
   - ui/logs
   - ui/tasks
   - ui/profile
   - ui/onboarding
   - ui/components
   - ui/theme
7. 创建 MainActivity 和 AppNavGraph。
8. 创建底部导航，占位页面包括：首页、专注、日志、任务、我的。
9. 首页先显示“口袋星球”和占位星球。
10. 基础主题要贴近参考图：浅蓝背景、奶油色卡片、深蓝标题、绿色选中态。

验收标准：

完成后请输出：
```

## 阶段 2 指令：生态规则与手动行为记录

```text
阶段目标：
实现行为记录到生态变化的最小闭环。

请完成：
1. 建立 Planet、DailyStats 的 Room Entity、Dao、Repository。
2. 实现 EcologyRuleEngine。
3. 首页或单独记录弹窗支持手动输入：
   - 步行分钟
   - 久坐分钟
   - 夜间活跃分钟
   - 通勤/外出分钟
4. 行为规则：
   - 步行：森林值增加 walkingMinutes * 0.8，荒漠值少量降低。
   - 久坐：荒漠值增加 sedentaryHours * 8。
   - 夜间活跃：暗影值增加 nightActiveMinutes * 1.2，梦境值降低。
   - 通勤/外出：城市值增加 commuteMinutes * 0.4。
5. 所有生态值必须限制在 0 到 100。
6. 每次行为记录后更新 DailyStats。
7. 首页实时显示最新生态值和状态。

验收标准：
1. 记录步行后森林值增加。
2. 记录久坐后荒漠值增加。
3. 记录夜间活跃后暗影值增加。
4. 记录通勤后城市值增加。
5. 数值不会超过 100 或低于 0。
6. 重启后数据不丢失。

完成后请输出：
1. 新增/修改文件列表。
2. 生态计算代码位置。
3. 手动测试步骤。
4. 已知问题。
```

## 阶段 3 指令：星球日志

```text
阶段目标：
实现行为触发星球事件日志。

请完成：
1. 建立 PlanetEvent Entity、Dao、Repository。
2. 实现 EventGenerator。
3. 满足以下条件时生成日志：
   - 步行 >= 30 分钟：森林降雨。
   - 专注 >= 60 分钟：水晶共鸣。
   - 久坐 >= 120 分钟：沙漠裂缝。
   - 夜间活跃 >= 60 分钟：暗影孢子。
4. 日志包含：
   - date
   - eventType
   - title
   - description
   - relatedValue
   - rarity
   - createdAt
5. 日志页按时间倒序展示。
6. 首页展示今日最新 1 到 3 条事件。
7. 日志页 UI 参考 docs/assets/ui-reference/logs.png，使用时间线样式，不要做成普通列表。

验收标准：
1. 行为达到条件后能生成日志。
2. 日志文案是星球世界观语言。
3. 重启后日志仍存在。
4. 首页能显示最新事件。

完成后请输出：
1. 新增/修改文件列表。
2. 事件生成规则位置。
3. 手动测试步骤。
4. 已知问题。
```

## 阶段 4 指令：专注计时

```text
阶段目标：
实现专注倒计时、专注记录、水晶奖励和日志。

请完成：
1. 建立 FocusSession Entity、Dao、Repository。
2. 专注页支持：
   - 25 分钟
   - 45 分钟
   - 自定义分钟
3. 支持开始、暂停、继续、结束。
4. 倒计时完成后：
   - 保存 completed = true 的 FocusSession。
   - 增加水晶值，奖励值等于专注分钟数，可按实际体验微调。
   - 更新 DailyStats.focusMinutes。
   - 生成水晶相关日志。
   - 更新首页生态状态。
5. 提前结束时：
   - 保存 completed = false 的 FocusSession。
   - 不给或少给奖励。
6. 专注页 UI 参考 docs/assets/ui-reference/focus.png：
   - 标题牌。
   - 大倒计时圆环。
   - 水晶塔或水晶岛视觉。
   - 开始、暂停、结束按钮。
   - 专注反馈卡片。

验收标准：
1. 倒计时可以正常开始、暂停、继续。
2. 完成后水晶值增加。
3. 完成后日志页出现专注事件。
4. 重启后专注记录不丢失。

完成后请输出：
1. 新增/修改文件列表。
2. 计时实现方式。
3. 手动测试步骤。
4. 后台计时限制说明。
```

## 阶段 5 指令：生态任务

```text
阶段目标：
实现每日任务和奖励领取。

请完成：
1. 建立 PlanetTask Entity、Dao、Repository。

5. 奖励领取后更新生态值并写入日志。
6. 已领取任务不能重复领取。
7. 任务页 UI 参考 docs/assets/ui-reference/tasks.png，使用大卡片、状态标签、进度条、奖励行。

验收标准：
1. 任务页能显示任务列表。
2. 任务进度能随行为变化。
3. 完成任务后可以领取奖励。
4. 重启后任务状态不丢失。
5. 同一任务不能重复领奖。

完成后请输出：
1. 新增/修改文件列表。
2. 任务生成规则位置。
3. 手动测试步骤。
```
