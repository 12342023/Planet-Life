# 口袋星球界面参考与 UI 实现规范

本文档基于 `docs/assets/ui-reference/` 中的 6 张界面参考图整理，用来指导 Gemini 在 Jetpack Compose 中还原整体视觉方向。MVP 不要求 100% 像素级复刻，但必须保持同一套视觉语言：浅蓝天空、像素风、奶油色面板、清晰底部导航、星球生态元素优先。

## 2. 总体视觉风格

关键词：像素风、治愈感、天空背景、奶油色卡片、蓝绿主调、生态养成、低压力游戏化。

所有主页面使用浅蓝天空背景。可以用纯色或轻微垂直渐变实现：顶部 `#8FD3F4`，底部 `#DDF7FF`。可以点缀少量白色云朵和星光，但 MVP 可先不做复杂背景装饰。

主要内容放在奶油白面板中：背景 `#FFF8E8` 或 `#FFFDF4`，边框 `#E7D5B7`，阴影轻柔，圆角约 16dp 到 28dp。后续可以用像素边框图片增强，MVP 先用圆角卡片。

文字层级建议：

- 页面标题：28sp 到 34sp，Bold，深蓝 `#31598F`。
- 卡片标题：20sp 到 24sp，Bold，深棕 `#4E4037`。
- 正文：15sp 到 18sp，Medium，深棕或灰棕。
- 数值：22sp 到 30sp，Bold，按生态类型着色。
- 底部导航文字：14sp 到 16sp，Medium。

## 4. 底部导航规范

参考图底部固定 5 个 Tab：

1. 星球
2. 日志
3. 专注
4. 任务
5. 我的

图鉴不放入 MVP 底部导航，建议从“我的”页面进入，或后续作为二级页面。

底部导航样式：

- 外层为奶油色圆角面板。
- 当前 Tab 使用浅绿色选中背景。
- 每个 Tab 包含图标和文字。
- 图标建议先用 emoji、简单矢量或 Compose 自绘占位，后续替换像素图标。

## 5. 主页面实现规范

参考：`home.png`

从上到下：

1. 顶部资料牌：星球图标、App 名称、星球名称、等级。
2. 中央大型星球岛。
3. 生态值卡片。
4. 今日事件卡片。
5. 底部导航。

参考图生态值展示五张卡片，但需求文档要求六类生态值。MVP 推荐使用 2 行 3 列，展示森林、水晶、梦境、城市、荒漠、暗影。每张卡包含图标、名称、数值、小进度条。

首页星球视觉可以先用 Compose Canvas 实现：根据最高生态值改变主色，根据荒漠值和暗影值增加沙色/紫色区域。验收重点是第一眼能看到星球，不能像普通健康数据面板。

## 6. 专注模式实现规范

参考：`focus.png`

从上到下：

1. 标题牌：专注模式。
2. 中央大倒计时圆环。
3. 水晶塔或水晶岛视觉。
4. 三个操作按钮：开始、暂停、结束。
5. 提示卡片：本次专注将为水晶塔增加能量。
6. 底部导航。

倒计时圆环占页面核心位置，中央显示 `25:00` 这类时间，圆环颜色使用水晶蓝到梦境紫。开始按钮浅绿，暂停按钮浅紫，结束按钮浅红。

## 7. 星球日志实现规范

参考：`logs.png`

从上到下：

1. 标题牌：今日星球日志。
2. 时间线卡片。
3. 日期切换区。
4. 底部导航。

每条日志包含左侧时间、时间线节点、像素图标或占位图标、事件文案。MVP 用 LazyColumn 实现，每条 item 高度保持一致。

示例：

- `07:20 晨光洒落，森林恢复生机`
- `09:10 今日步行达成，森林值 +5`
- `14:00 专注 30 分钟，水晶塔能量上升`
- `23:50 昨夜晚睡，暗影蘑菇悄悄生长`

## 8. 生态任务实现规范

参考：`tasks.png`

任务页从上到下是标题牌、任务卡片列表、底部导航。每张任务卡包含左侧大图标、任务名、目标说明、状态标签、进度条、当前进度、奖励说明。

状态标签：

- 进行中：浅蓝。
- 已完成：浅绿。
- 待开始：浅紫。

参考任务：

- 让南部森林下雨：目标今日步行 30 分钟。
- 点亮水晶塔：目标专注 25 分钟。
- 守护梦境云层：目标今晚 23:30 前入睡。
- 净化荒漠边缘：目标久坐不超过 60 分钟。

## 9. 星球图鉴实现规范

参考：`collection.png`

图鉴页从上到下是标题牌、分类切换、生物/地貌/事件两列网格。已解锁卡片显示名称、图标、简短描述、稀有度星光；未解锁卡片显示锁图标、`???`、剪影、解锁条件。

参考图中的生物：

- 云眠兔：栖息在梦境云层，喜欢安静的夜晚。
- 晶簇精灵：在水晶塔附近出现，会回应专注能量。
- 晨露蝶：清晨出现，为森林带来活力。
- 暗影蘑菇：在夜深时悄悄生长，喜欢阴影。

需求文档中的森林鹿、水晶猫、睡眠水母等可以与参考图生物合并或替换。为了贴合参考图，MVP 可优先使用云眠兔、晶簇精灵、晨露蝶、暗影蘑菇。

## 10. 我的页面实现规范

参考：`profile.png`

从上到下：

1. 标题牌：我的。
2. 用户资料大卡片。
3. 统计卡片。
4. 设置列表。
5. 底部导航。

用户资料大卡片显示头像、昵称、编辑按钮、小星球图、星球名称、星球等级、图鉴解锁进度。统计卡片显示累计专注时间和累计步行时间。设置项至少包含主题切换、提醒设置、隐私设置，后续增加目标设置、清空本地数据、关于应用。

## 11. Gemini UI 开发要求

给 Gemini 的 UI 指令中必须加入：

```text
请参考 docs/assets/ui-reference 目录下的界面图，整体风格为浅蓝天空背景、像素风、奶油色面板、治愈系星球养成。

MVP 可以先用 Compose Canvas、emoji、简单图标或占位图实现，不要求立即制作完整像素素材，但布局层级和视觉气质要接近参考图。

底部导航固定为：星球、日志、专注、任务、我的。

星球图鉴从“我的”页面进入，或者作为二级页面，不作为第一版底部导航。
```

## 12. UI 验收清单

- 所有主页面都有浅蓝天空背景。
- 所有主页面标题使用奶油色标题牌。
- 底部导航为 5 个 Tab。
- 当前 Tab 有浅绿色选中状态。
- 首页以星球视觉为主，不以数据表为主。
- 生态值有图标、数值、进度条。
- 专注页中央是大倒计时圆环。
- 日志页是时间线，不是普通列表。
- 任务页是大卡片和进度条。
- 图鉴页是两列网格，有已解锁和未解锁状态。
- 我的页包含资料大卡、统计卡、设置项。
- 文案保持星球世界观。
