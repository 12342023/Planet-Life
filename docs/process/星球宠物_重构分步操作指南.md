# 星球宠物 重构分步操作指南

版本：V1.0  
日期：2026-05-13  
用途：把《星球宠物》重构方案拆成小步骤，供 Codex 给 Gemini 下达任务、审查结果、验证并记录。

## 1. 总原则

本重构必须遵守：

> 每一步只做一件小事。  
> 每一步走通后，才能进入下一步。

任何阶段都不能一次性大改多个系统。  
如果某一步没有通过构建、启动、审查或人工验收，必须先修复这一步，不能继续往后做。

## 2. 每一步固定流程

每一步都按同一个流程执行。

### 2.1 Codex 先检查

开始前先检查：

```text
git status --short
./gradlew assembleDebug
```

如果涉及 UI 或启动流程，还要安装启动：

```text
/Users/jianghao/Library/Android/sdk/platform-tools/adb install -r app/build/outputs/apk/debug/app-debug.apk
/Users/jianghao/Library/Android/sdk/platform-tools/adb shell am start -W -n com.example.planetlife/.MainActivity
/Users/jianghao/Library/Android/sdk/platform-tools/adb logcat -d -v time -s AndroidRuntime ActivityTaskManager
```

### 2.2 Codex 给 Gemini 下命令

每次给 Gemini 的命令必须包含：

```text
目标：
范围：
禁止：
需要读取的文件：
需要修改的文件：
验收标准：
完成后请说明：
```

禁止 Gemini 自己跨阶段扩展功能。

### 2.3 Gemini 实现

Gemini 只实现本步骤内容。

不能顺手做：

- 3D。
- AI。
- Health Connect。
- 大规模 UI 重构。
- 数据库大改。
- 额外页面。
- 不在本步骤范围内的优化。

### 2.4 Codex 审查

Gemini 完成后，Codex 检查：

```text
git diff
rg 关键字
./gradlew assembleDebug
```

如果涉及 UI：

- 安装 APK。
- 启动 App。
- 查看 logcat。
- 手动走一遍该功能。

### 2.5 记录与提交

通过后记录：

- 新增文件。
- 修改文件。
- 验证命令。
- 已知问题。
- 下一步。

然后提交：

```text
git add ...
git commit -m "..."
git push origin main
```

## 43. 停止条件

遇到以下情况必须停止，不进入下一步：

- `./gradlew assembleDebug` 失败。
- App 启动闪退。
- 数据库 migration 风险不清楚。
- UI 明显重叠或无法点击。
- 心情天气出现惩罚性逻辑。
- AI 文案出现诊断、指责、焦虑语言。
- Gemini 修改了本阶段范围外的大量文件。
